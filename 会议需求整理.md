# 会议需求整理

## 一、会议核心背景

本次会议针对 NDR2.0 版本的需求分析与推进展开，当前核心问题为：

1. 2.0 版本需求完成时间紧迫，团队成员工作负荷较大

2. 应对方案：

    - 按模块拆分需求，由不同成员分别负责

    - 建立沟通机制：每周内部进度沟通，阶段性向领导汇报

---

## 二、核心需求分类

### 1. 方向性需求（核心业务方向）

|需求项|具体说明|
|---|---|
|沙箱补充|完成本地沙箱和云沙箱功能开发，作为安全检测的隔离环境|
|加密流量分析|增强加密流量分析能力，针对加密通信进行安全监测|
|挖矿检测|作为优先商业机会点，识别非法加密货币挖矿活动|
|AI 安全监测|新增 AI 监测专题场景，建立针对 AI 系统自身风险的检测机制|
### 2. 能力性需求（平台能力升级）

|需求项|具体说明|
|---|---|
|平台优化|完善平台功能，提升产品易用性|
|两级平台架构|实现集中管理多探针设备，统一查看状态 / 版本，支持一键策略下发；逐步向态势感知演进|
|AI 狩猎|通过 AI 模型辅助告警确认、攻击路径分析，实现威胁追踪，覆盖反诈 / 挖矿 / NDR / 零信任等场景|
|溯源能力|实现类似 "黄金眼" 的 IP / 端口溯源，自动追踪攻击路径，还原完整证据链|
### 3. 扩展类需求（能力扩展集成）

|需求项|具体说明|
|---|---|
|零信任对接|集成零信任安全能力，配合现有项目需求完成对接|
|数据安全检测|聚焦高价值典型场景，扩展 API 安全、数据库审计等功能，采用渐进式集成方案|
|第三方数据联动|支持第三方数据接入，可实现 NDR 网络攻击数据与数据安全告警的关联分析|
### 4. 性能类需求（底层性能改造）

|需求项|具体说明|
|---|---|
|数据库改造|将数据库迁移到 ES-CK 架构，完成性能升级|
|系统性能提升|整体优化系统运行性能，支撑更大规模的检测与分析|
### 5. 规则类需求（规则效能优化）

|需求项|具体说明|
|---|---|
|规则命中率统计|通过 ID 关联统计规则命中率与使用率，定时执行|
|规则优化机制|对长期零命中的规则进行关闭优化，持续验证现有规则有效性|
